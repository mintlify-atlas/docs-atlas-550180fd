---
title: "Create Pet with Image"
api: "POST /api/pets/withimage"
description: "Create a new pet with an image upload"
---

This endpoint creates a new pet in the adoption system with an uploaded image. The image is processed using Multer middleware and stored in the server's file system.

### Body

This endpoint accepts `multipart/form-data` content type for file uploads.

<ParamField body="name" type="string" required>
  The name of the pet.
</ParamField>

<ParamField body="specie" type="string" required>
  The species of the pet (e.g., "dog", "cat", "bird", "rabbit").
</ParamField>

<ParamField body="birthDate" type="string" required>
  The birth date of the pet. Can be in ISO 8601 format or any valid date string.
</ParamField>

<ParamField body="image" type="file" required>
  The image file for the pet. Uploaded using the `image` field name.
  
  The file is processed by Multer middleware with the following configuration:
  - **Storage location**: `/public/img/` directory
  - **Filename format**: `{timestamp}-{originalFilename}` (e.g., `1678901234567-pet-photo.jpg`)
  - **Field name**: Must be `image` (matched by `uploader.single('image')`)
</ParamField>

### Response

<ResponseField name="status" type="string">
  The status of the response. Returns `"success"` on successful creation.
</ResponseField>

<ResponseField name="payload" type="object">
  The created pet object.
  
  <ResponseField name="_id" type="string">
    The unique identifier for the newly created pet.
  </ResponseField>
  
  <ResponseField name="name" type="string">
    The name of the pet.
  </ResponseField>
  
  <ResponseField name="specie" type="string">
    The species of the pet.
  </ResponseField>
  
  <ResponseField name="birthDate" type="string">
    The birth date of the pet in ISO 8601 format.
  </ResponseField>
  
  <ResponseField name="adopted" type="boolean">
    Whether the pet has been adopted. Defaults to `false` for new pets.
  </ResponseField>
  
  <ResponseField name="image" type="string">
    The server file path where the uploaded image is stored.
  </ResponseField>
</ResponseField>

## Implementation Details

### Multer Configuration

The image upload is handled by Multer middleware configured in `/src/utils/uploader.js`:

```javascript
const storage = multer.diskStorage({
  destination: function(req, file, cb) {
    cb(null, `${__dirname}/../public/img`)
  },
  filename: function(req, file, cb) {
    cb(null, `${Date.now()}-${file.originalname}`)
  }
})

const uploader = multer({storage})
```

- Files are stored in the `/public/img/` directory
- Filenames are prefixed with a timestamp to ensure uniqueness
- The middleware is applied using `uploader.single('image')` in the route definition
