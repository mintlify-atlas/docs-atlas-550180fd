## Introduction

The Adoptme API is a RESTful service for managing pet adoptions. It provides endpoints for managing users, pets, adoptions, and authentication sessions.

## Base URL

The API runs on a configurable port (default: 8080):

```
http://localhost:8080/api
```

<Note>
The base URL can be configured via the `PORT` environment variable. In production, replace `localhost:8080` with your deployed server URL.
</Note>

## API Documentation

Interactive API documentation is available via Swagger UI at:

```
http://localhost:8080/docs
```

The Swagger interface provides an interactive way to explore and test all API endpoints directly from your browser.

## Authentication

The Adoptme API uses **JWT (JSON Web Token)** authentication with HTTP-only cookies.

### Authentication Flow

1. **Register** a new user at `/api/sessions/register`
2. **Login** at `/api/sessions/login` to receive a JWT token
3. The token is automatically stored in a cookie named `coderCookie`
4. Include the cookie in subsequent requests to protected endpoints

### Token Details

- **Cookie Name**: `coderCookie`
- **Token Type**: JWT
- **Expiration**: 1 hour
- **Secret**: `tokenSecretJWT` (configurable)

<Warning>
Tokens expire after 1 hour. Users must re-authenticate when their token expires.
</Warning>

### Example Authentication

<CodeGroup>

```bash Register
curl -X POST http://localhost:8080/api/sessions/register \
  -H "Content-Type: application/json" \
  -d '{
    "first_name": "John",
    "last_name": "Doe",
    "email": "john.doe@example.com",
    "password": "securePassword123"
  }'
```

```bash Login
curl -X POST http://localhost:8080/api/sessions/login \
  -H "Content-Type: application/json" \
  -c cookies.txt \
  -d '{
    "email": "john.doe@example.com",
    "password": "securePassword123"
  }'
```

```bash Authenticated Request
curl -X GET http://localhost:8080/api/sessions/current \
  -b cookies.txt
```

</CodeGroup>

## Request Format

### Headers

All requests should include:

```
Content-Type: application/json
```

For authenticated requests, the JWT cookie is automatically sent by the browser or HTTP client.

### Request Body

Request bodies should be formatted as JSON:

```json
{
  "first_name": "Jane",
  "last_name": "Smith",
  "email": "jane@example.com",
  "password": "password123"
}
```

## Response Format

All API responses follow a consistent format with a `status` field and additional data.

### Success Response

```json
{
  "status": "success",
  "payload": {
    "id": "6893eaba2ac0b16fa177be7a",
    "first_name": "Jerónimo",
    "last_name": "Arroyo Alonzo",
    "email": "Jeronimo_ArroyoAlonzo63@gmail.com",
    "role": "user",
    "pets": []
  }
}
```

### Error Response

```json
{
  "status": "error",
  "error": "User not found"
}
```

<Info>
The `status` field will be either `"success"` or `"error"` to indicate the result of the operation.
</Info>

## HTTP Status Codes

The API uses standard HTTP status codes:

| Code | Description | Usage |
|------|-------------|-------|
| `200` | OK | Successful GET, PUT, or DELETE request |
| `201` | Created | Successful POST request that created a resource |
| `400` | Bad Request | Invalid request format or missing required fields |
| `404` | Not Found | Requested resource does not exist |
| `500` | Internal Server Error | Server-side error occurred |

### Common Error Scenarios

#### 400 Bad Request

Returned when required fields are missing or invalid:

```json
{
  "status": "error",
  "error": "Incomplete values"
}
```

#### 404 Not Found

Returned when a resource doesn't exist:

```json
{
  "status": "error",
  "error": "User not found"
}
```

#### 500 Internal Server Error

Returned when an unexpected server error occurs:

```
Ha ocurrido un error en la petición, por favor ver detalle: [error details]
```

## Rate Limiting

<Info>
Currently, the API does not implement rate limiting. All authenticated users have unlimited access to endpoints.
</Info>

## API Resources

The Adoptme API provides the following main resources:

- **[Users](/api-reference/users)** - Manage user accounts
- **[Pets](/api-reference/pets)** - Browse and manage pets available for adoption
- **[Adoptions](/api-reference/adoptions)** - Create and manage adoption records
- **[Sessions](/api-reference/sessions)** - Authentication and session management
- **[Mocks](/api-reference/mocks)** - Generate test data for development

## Database

The API connects to MongoDB with configurable connection settings:

- **Default MongoDB URL**: `mongodb://127.0.0.1:27017`
- **Default Database Name**: `adoptme`

<Note>
Database connection settings can be configured via `MONGO_URL` and `DB_NAME` environment variables.
</Note>
