---
title: Unprotected login
description: Authenticate user with unprotected cookie (for testing/development)
api: GET /api/sessions/unprotectedLogin
---

Authenticate a user and receive an unprotected JWT token. This endpoint is similar to the standard login but stores the full user object (not a DTO) in the JWT token and uses a different cookie name.

<Warning>
  This endpoint is intended for testing and development purposes only. It exposes the full user object including the hashed password in the JWT payload. Use the standard `/api/sessions/login` endpoint in production.
</Warning>

## Request Body

<ParamField body="email" type="string" required>
  The user's email address
</ParamField>

<ParamField body="password" type="string" required>
  The user's password (will be verified against bcrypt hash)
</ParamField>

## Response

<ResponseField name="status" type="string">
  Response status - "success" or "error"
</ResponseField>

<ResponseField name="message" type="string">
  Success message: "Unprotected Logged in"
</ResponseField>

The JWT token is set as an HTTP-only cookie named `unprotectedCookie` with:
- **Max Age**: 3600000ms (1 hour)
- **Payload**: Full user object from database (including all fields)
- **Secret**: 'tokenSecretJWT'
- **Expiration**: 1 hour

### Success Response (200 OK)

```json
{
  "status": "success",
  "message": "Unprotected Logged in"
}
```

### Error Responses

**400 Bad Request** - Missing credentials
```json
{
  "status": "error",
  "error": "Incomplete values"
}
```

**400 Bad Request** - Invalid password
```json
{
  "status": "error",
  "error": "Incorrect password"
}
```

**404 Not Found** - User doesn't exist
```json
{
  "status": "error",
  "error": "User doesn't exist"
}
```

**500 Internal Server Error**
```
Ha ocurrido un error en la petición, por favor ver detalle: [error details]
```

## Example Request

<CodeGroup>
```bash cURL
curl -X GET http://localhost:8080/api/sessions/unprotectedLogin \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john@example.com",
    "password": "password123"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:8080/api/sessions/unprotectedLogin', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json',
  },
  credentials: 'include', // Important: include cookies
  body: JSON.stringify({
    email: 'john@example.com',
    password: 'password123'
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.get(
    'http://localhost:8080/api/sessions/unprotectedLogin',
    json={
        'email': 'john@example.com',
        'password': 'password123'
    }
)

print(response.json())
```
</CodeGroup>

## Key Differences from Standard Login

| Feature | Standard Login | Unprotected Login |
|---------|---------------|-------------------|
| Cookie Name | `coderCookie` | `unprotectedCookie` |
| JWT Payload | UserDTO (name, role, email) | Full user object |
| Password in Payload | No | Yes (hashed) |
| Production Use | ✅ Yes | ❌ No - Testing only |
