## Overview

The Adoptme API is fully containerized using Docker, making it easy to deploy and run in any environment. The Docker image is based on Node.js 20 and includes all necessary dependencies.

## Dockerfile

The project includes a production-ready Dockerfile:

```dockerfile Dockerfile
FROM node:20.11.0

WORKDIR /app

COPY package*.json ./

RUN npm install

COPY ./src ./src

EXPOSE 3000

CMD ["npm","start"]
```

### Dockerfile Breakdown

- **Base Image**: Uses Node.js 20.11.0 for optimal performance and compatibility
- **Working Directory**: Sets `/app` as the container's working directory
- **Dependencies**: Copies package files and installs dependencies before copying source code (leverages Docker layer caching)
- **Source Code**: Only copies the `src` directory to keep the image size minimal
- **Port**: Exposes port 3000 (configurable via environment variables)
- **Command**: Starts the application using `npm start`

<Note>
  The Dockerfile uses a multi-step copy process to optimize build times. Dependencies are installed first, allowing Docker to cache this layer and only rebuild when `package.json` changes.
</Note>

## Building the Docker Image

<Steps>
  <Step title="Navigate to project directory">
    ```bash
    cd /path/to/adoptme-api
    ```
  </Step>

  <Step title="Build the image">
    Build the Docker image with a tag:
    
    ```bash
    docker build -t DOCKER_USERNAME/serverxxxx:1.0.0 .
    ```
    
    Replace `DOCKER_USERNAME` with your Docker Hub username and `serverxxxx` with your desired image name.
  </Step>

  <Step title="Verify the build">
    Check that the image was created successfully:
    
    ```bash
    docker images
    ```
    
    You should see your image listed with the specified tag.
  </Step>
</Steps>

<CodeGroup>
```bash Example Build Command
docker build -t fabil2025/serverentregafinal:1.0.0 .
```

```bash Build with latest tag
docker build -t myusername/adoptme-api:latest .
```

```bash Build with custom name
docker build -t adoptme-production:v1.0.0 .
```
</CodeGroup>

## Running the Docker Container

### Using Docker CLI

<Steps>
  <Step title="Run the container">
    ```bash
    docker run -d \
      --name adoptme-container \
      -p 8080:3000 \
      -e PORT=3000 \
      -e MONGO_URL=your_mongodb_connection_string \
      -e DB_NAME=adoptme \
      your-image-name:tag
    ```
  </Step>

  <Step title="Verify container is running">
    ```bash
    docker ps
    ```
  </Step>

  <Step title="View container logs">
    ```bash
    docker logs adoptme-container
    ```
  </Step>
</Steps>

### Using Docker Desktop

<Steps>
  <Step title="Locate your image">
    Open Docker Desktop and navigate to the Images tab. Find your built image in the list.
  </Step>

  <Step title="Click the Play button">
    Click the Play button next to your image to open the container configuration screen.
  </Step>

  <Step title="Configure container settings">
    Fill in the required configuration:
    
    **Container Name**: Choose a descriptive name (e.g., `adoptme-api`)
    
    **Ports**: Map host port to container port (e.g., `8080:3000`)
    
    **Environment Variables**: Add the required variables:
    ```
    PORT=3000
    MONGO_URL=your_mongodb_connection_string
    DB_NAME=adoptme
    ```
  </Step>

  <Step title="Start the container">
    Click "Run" to start the container. You should see it appear in the Containers tab with a green "Running" status.
  </Step>
</Steps>

<Warning>
  The application requires valid `MONGO_URL` and `DB_NAME` environment variables to start. Without these, the container will fail to initialize as it attempts to connect to MongoDB on startup.
</Warning>

## Environment Variables

The Docker container requires the following environment variables:

| Variable | Required | Description | Example |
|----------|----------|-------------|----------|
| `PORT` | Yes | Port number for the API server | `3000` |
| `MONGO_URL` | Yes | MongoDB connection string | `mongodb://localhost:27017` |
| `DB_NAME` | Yes | MongoDB database name | `adoptme` |

<Note>
  If you don't provide `MONGO_URL` and `DB_NAME`, the application will fail to start because it attempts to connect to MongoDB during initialization.
</Note>

## Docker Compose (Optional)

For easier container management, you can create a `docker-compose.yml` file:

```yaml docker-compose.yml
version: '3.8'

services:
  adoptme-api:
    image: fabil2025/serverentregafinal:1.0.0
    container_name: adoptme-api
    ports:
      - "8080:3000"
    environment:
      - PORT=3000
      - MONGO_URL=mongodb://mongo:27017
      - DB_NAME=adoptme
    depends_on:
      - mongo
    restart: unless-stopped

  mongo:
    image: mongo:6
    container_name: adoptme-mongo
    ports:
      - "27017:27017"
    volumes:
      - mongo-data:/data/db
    restart: unless-stopped

volumes:
  mongo-data:
```

Run with:

```bash
docker-compose up -d
```

## Troubleshooting

<Accordion title="Container fails to start">
  **Symptom**: Container starts and immediately stops
  
  **Solution**: Check the logs for error messages:
  ```bash
  docker logs adoptme-container
  ```
  
  Common causes:
  - Missing or invalid `MONGO_URL`
  - Missing or invalid `DB_NAME`
  - MongoDB server not accessible
  - Port already in use on the host
</Accordion>

<Accordion title="Cannot connect to MongoDB">
  **Symptom**: Application logs show MongoDB connection errors
  
  **Solution**: Verify your MongoDB connection string:
  - Ensure MongoDB is running and accessible
  - Check network connectivity between containers
  - Verify authentication credentials if using MongoDB Atlas
  - Use `host.docker.internal` instead of `localhost` when connecting to MongoDB on the host machine
</Accordion>

<Accordion title="Port conflicts">
  **Symptom**: Error message about port already in use
  
  **Solution**: Use a different host port:
  ```bash
  docker run -p 8081:3000 ...
  ```
</Accordion>

<Accordion title="Image build fails">
  **Symptom**: Docker build command fails with errors
  
  **Solution**:
  - Ensure you're in the project root directory
  - Check that `package.json` and `src/` directory exist
  - Verify Docker daemon is running
  - Check for sufficient disk space
</Accordion>

## Best Practices

1. **Use specific version tags** instead of `latest` for production deployments
2. **Set resource limits** to prevent containers from consuming excessive resources:
   ```bash
   docker run --memory="512m" --cpus="1.0" ...
   ```
3. **Use Docker secrets** for sensitive environment variables in production
4. **Enable health checks** to monitor container status
5. **Use multi-stage builds** to reduce image size (future improvement)
6. **Regularly update base images** to get security patches
