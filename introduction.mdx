# Introduction to Adoptme API

Adoptme is a RESTful API designed to manage pet adoption processes. Built with modern Node.js technologies, it provides a complete backend solution for pet adoption platforms, including user management, pet listings, and adoption tracking.

## What is Adoptme?

Adoptme API is a production-ready backend system that enables you to:

- **Manage Users**: Register, authenticate, and manage user profiles with role-based access control
- **Pet Listings**: Create, update, and manage pet profiles with image uploads
- **Adoption Tracking**: Track and manage the adoption process between users and pets
- **Mock Data Generation**: Generate realistic test data for development and testing
- **API Documentation**: Built-in Swagger documentation for all endpoints

## Technology Stack

The Adoptme API is built with modern, production-tested technologies:

- **Runtime**: Node.js 22+
- **Framework**: Express.js 4.18+
- **Database**: MongoDB with Mongoose ODM
- **Authentication**: JWT tokens with bcrypt password hashing
- **Documentation**: Swagger UI with OpenAPI 3.0
- **Testing**: Mocha, Chai, and Supertest
- **Mock Data**: Faker.js for realistic test data

## API Endpoints

The API is organized into five main resource groups:

### Users (`/api/users`)
Manage user accounts, profiles, and user data.

### Pets (`/api/pets`)
Create and manage pet listings with detailed information and images.

### Adoptions (`/api/adoptions`)
Track and manage the adoption process.

### Sessions (`/api/sessions`)
Handle user authentication, registration, and session management.

### Mocks (`/api/mocks`)
Generate mock data for testing and development purposes.

## Architecture Overview

The Adoptme API follows a layered architecture pattern:

```
src/
├── app.js                  # Application entry point
├── config/                 # Configuration management
├── routes/                 # API route definitions
├── controllers/            # Request handlers
├── services/               # Business logic layer
├── dao/                    # Data Access Objects
├── repository/             # Repository pattern implementation
├── dto/                    # Data Transfer Objects
├── models/                 # Mongoose schemas
└── utils/                  # Utility functions
```

<Note>
This architecture separates concerns and makes the codebase maintainable, testable, and scalable.
</Note>
