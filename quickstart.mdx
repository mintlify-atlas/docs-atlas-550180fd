# Quickstart Guide

Get up and running with Adoptme API in just a few minutes. This guide will walk you through the essentials to make your first API call.

## Prerequisites

Before you begin, ensure you have:

<CardGroup cols={2}>
  <Card title="Node.js 22+" icon="node-js">
    Check your version with `node --version`
  </Card>
  <Card title="MongoDB" icon="database">
    Local instance or MongoDB Atlas connection
  </Card>
</CardGroup>

<Note>
**Check your Node.js version**: Run `node --version` in your terminal. If you have an older version, download Node.js 22 or higher from [nodejs.org](https://nodejs.org/).
</Note>

## Quick Setup

<Steps>
  <Step title="Clone the Repository">
    Clone the Adoptme API repository to your local machine:

    ```bash
    git clone https://github.com/GitRepoFabi/coderhouse-backendIII.git .
    ```
  </Step>

  <Step title="Install Dependencies">
    Install all required npm packages:

    ```bash
    npm install
    ```

    This will install all dependencies including:
    - Express.js for the web framework
    - Mongoose for MongoDB interactions
    - JWT and bcrypt for authentication
    - Swagger for API documentation
  </Step>

  <Step title="Configure Environment Variables">
    Create a `.env` file in the root directory with your configuration:

    ```bash
    PORT=8080
    MONGO_URL=mongodb://127.0.0.1:27017
    DB_NAME=adoptme
    ```

    <Accordion title="Environment Variables Explained">
      - **PORT**: The port number where the server will run (default: 8080)
      - **MONGO_URL**: Your MongoDB connection string
      - **DB_NAME**: The name of the database to use (default: adoptme)

      <Note>
      If you're using MongoDB Atlas, your `MONGO_URL` will look like:
      ```
      mongodb+srv://username:password@cluster.mongodb.net/
      ```
      </Note>
    </Accordion>
  </Step>

  <Step title="Start the Server">
    Launch the API server:

    ```bash
    npm start
    ```

    You should see:
    ```
    Connection successful MongoDB
    Listening on 8080
    ```

    <Note>
    For development with auto-reload, use `npm run dev` instead. This uses nodemon to automatically restart the server when you make changes.
    </Note>
  </Step>
</Steps>

## Your First API Call

Now that the server is running, let's make your first API call!

### Get All Pets

Fetch all pets from the database:

<CodeGroup>
```bash cURL
curl http://localhost:8080/api/pets
```

```javascript JavaScript (Fetch)
const response = await fetch('http://localhost:8080/api/pets');
const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.get('http://localhost:8080/api/pets')
data = response.json()
print(data)
```
</CodeGroup>

**Response:**
```json
{
  "status": "success",
  "payload": []
}
```

### Create a Pet

Add a new pet to the database:

<CodeGroup>
```bash cURL
curl -X POST http://localhost:8080/api/pets \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Max",
    "specie": "Dog",
    "birthDate": "2020-05-15"
  }'
```

```javascript JavaScript (Fetch)
const response = await fetch('http://localhost:8080/api/pets', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'Max',
    specie: 'Dog',
    birthDate: '2020-05-15'
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests
import json

payload = {
    "name": "Max",
    "specie": "Dog",
    "birthDate": "2020-05-15"
}

response = requests.post(
    'http://localhost:8080/api/pets',
    headers={'Content-Type': 'application/json'},
    data=json.dumps(payload)
)

data = response.json()
print(data)
```
</CodeGroup>

**Response:**
```json
{
  "status": "success",
  "payload": {
    "_id": "507f1f77bcf86cd799439011",
    "name": "Max",
    "specie": "Dog",
    "birthDate": "2020-05-15T00:00:00.000Z",
    "adopted": false
  }
}
```

### Register a User

Create a new user account:

<CodeGroup>
```bash cURL
curl -X POST http://localhost:8080/api/sessions/register \
  -H "Content-Type: application/json" \
  -d '{
    "first_name": "John",
    "last_name": "Doe",
    "email": "john.doe@example.com",
    "password": "securepassword123"
  }'
```

```javascript JavaScript (Fetch)
const response = await fetch('http://localhost:8080/api/sessions/register', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    first_name: 'John',
    last_name: 'Doe',
    email: 'john.doe@example.com',
    password: 'securepassword123'
  })
});

const data = await response.json();
console.log(data);
```
</CodeGroup>

## Troubleshooting

<Accordion title="Port already in use">
  If port 8080 is already in use, change the `PORT` value in your `.env` file:
  ```bash
  PORT=3000
  ```
</Accordion>

<Accordion title="MongoDB connection failed">
  Ensure MongoDB is running:
  ```bash
  # For local MongoDB
  sudo systemctl status mongod
  
  # Or check if MongoDB process is running
  ps aux | grep mongod
  ```
  
  Verify your `MONGO_URL` in the `.env` file is correct.
</Accordion>

<Accordion title="Node version error">
  If you see module or syntax errors, verify your Node.js version:
  ```bash
  node --version
  ```
  
  Adoptme requires Node.js 22 or higher. Update if necessary.
</Accordion>
